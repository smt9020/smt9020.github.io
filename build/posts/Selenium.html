Selenium - what makes a good framework?

I'm sure there is a lot more to it then this, but over time I hope to compile a list of things you need for a good Selenium framework. "Good" is going to mean something different to everyone, and is going to depend a lot on who your audience is. In many projects, your audience is the developers who want to be able to test basic functionality of their web app over and over without having to do it manually. It's also common to see selenium wrapped in continuous integration solutions like Jenkins or TeamCity. My audience in this case is the QA group. How can we create a maintainable group of tests that can be run and maintained during regression testing by Test Engineers who do not have experience with coding? There are several building blocks for creating a Selenium framework that BOTH soltutions may find useful.

  • Test Input

    • A way of structuring the parameters of your test so that you can run the same test over and over with different boundary cases. If your audiences is mostly tech-saavy developers, you can do this with command line arguments. If they are not, you can do this with a custom runner, or supplying a spreadsheet that your test can parse. Alternatively you can create a custom connecttion of your Selenium framework to test management solutions such as HP QC, and use the parameter support that comes with that.

  • Object Repository

    • Organizing your page objects so that you can reference them with a friendly-name, and also only have to make a change in one place when a UI change will alter your test or break it. This can be as simple as a class of constants with xpaths to identify different buttons on a screen. It could also take the form of a spreadsheet.

  • Assertions and Reporting

    • using NUnit or JUnit for assertions is a pretty easy win. And then you can create your own logging, or get really fancy and use something like log4net.

  • Libraries

    • Broken down by page = Useful more when your audience is development

    Abstract with methods such as "click element" = Useful more when your audience is QA

  • Version Control

    • Obviously you need to version control your Selenium code. But what about controlling which versions of the test work on which versions of the product you are testing? You need to have a way of versioning your object repository and your task flow based on which version of the product that test will work on.

  • Batch Runner

    • Apache ANT for batch execution on many manual testers machines.
